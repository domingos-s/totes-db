# Totes.DB

Totes.DB is a static progressive web app for creating QR-code labels tied to garage tote inventory records.

## Features

- Create, edit, delete tote records
- Fields:
  - QR code / Tote ID
  - Tote title
  - Tote contents
  - Typical access season
- Generate QR codes for each tote
- Print all labels or a single tote label
- Search and filter records
- Export/import records as JSON
- Offline-first PWA with service worker caching
- Installable on Android and iOS
- SVG app icons included

## How to run locally

Because service workers require `http://localhost` or HTTPS, do not just double-click `index.html`.

From the app folder, run:

```bash
python3 -m http.server 8080
```

Then open:

```text
http://localhost:8080
```

## How to install on Android

1. Open the hosted app in Chrome.
2. Tap the install prompt or Chrome menu.
3. Select **Add to Home screen** or **Install app**.

## How to install on iPhone / iPad

1. Open the hosted app in Safari.
2. Tap the Share button.
3. Tap **Add to Home Screen**.
4. Confirm the app name: **Totes.DB**.

## Hosting options

This app can be hosted on any static file host, including:

- GitHub Pages
- Netlify
- Vercel
- Cloudflare Pages
- Any basic web server

## Important data note

Records are stored in the browser's `localStorage` on each device. Use **Export JSON** periodically as a backup or to move records between devices.

## Label printing

The print layout is designed for generic 2-column labels approximately 3.75" × 2.25". You can adjust the `@media print` section in `styles.css` if you use different label stock.


## GitHub Pages deployment

This package is already GitHub Pages-friendly because all app paths are relative.

Recommended setup:

1. Create a new GitHub repository, for example: `totes-db`.
2. Upload all files from this folder to the repository root.
3. Go to **Settings → Pages**.
4. Under **Build and deployment**, choose:
   - Source: **Deploy from a branch**
   - Branch: `main`
   - Folder: `/root`
5. Save.
6. Open the published GitHub Pages URL.

Example URL format:

```text
https://YOUR-GITHUB-USERNAME.github.io/totes-db/
```

### Important QR behavior on GitHub Pages

QR codes generated by the app point to the current hosted app URL plus the tote record hash.

Example:

```text
https://YOUR-GITHUB-USERNAME.github.io/totes-db/#tote=TOTE-ABCD-1234
```

Because records are stored in each device's browser `localStorage`, scanning a QR code only opens the record on a device/browser where the record data exists. Use **Export JSON** and **Import JSON** to move the tote database between devices.

### Best practice

After publishing to GitHub Pages, create/print your final labels from the live GitHub Pages URL, not from `localhost`, so the QR labels point to the permanent hosted app.
